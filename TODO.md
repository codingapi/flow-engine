# 流程开发

在节点设置上支持的功能如下：
1. 流程节点设置支持多人办理设置：按顺序执行、会签（设置比例）、或签、随机人员审批
2. 拒绝时的流程配置，退回上级节点、退回指定节点、终止流程
3. 审批人与提交人时的控制，由自己审批、自动跳过
4. 审批意见是否必填的控制
5. 是否需要签名的控制，开启以后在审批时需要填写签名
6. 超时处理机制，超时以后自动处理流程，设定超时时间，执行方式：自动提醒、自动同意、自动拒绝

# 流程支持的操作：
1. 通过，流程继续往下流转
2. 拒绝，拒绝时需要根据拒绝的配置流程来设置
3. 加签，指定给其他人一块审批，以会签模式来处理（需要设定人员范围）
4. 委派，委派给其他人员来审批，当人员审批完成以后再流程给自己审批（需要设定人员范围）
5. 退回，退回时需要设置退回的节点
6. 转办，将流程转移给指定用户来审批，需要配置人员匹配范围（需要设定人员范围）
7. 撤销，再流程已经办理之后当下一个审批人员尚未办理之前可以撤回流程
8. 自定义，支持用户根据需求自定义配置的流程按钮操作能力

委托人员支持
在审批人请假或不在岗时，可以将审批权限委托给他人。
