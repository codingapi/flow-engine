Flow Engine 使用手册

本手册从前端功能操作视角介绍 Flow Engine 的能力与使用方式，面向业务用户、前端开发与实施人员。内容涵盖 PC 端首页导航、流程设计、待办中心、流程审批，以及移动端的待办与审批操作，所有界面均配以真实截图说明。

本手册基于官方示例应用 app-pc（PC 端）与 app-mobile（移动端）描述操作。后端需已按 集成手册 完成集成并启动，截图存放于 docs/manual/screenshot/ 目录。

1. 系统能力总览

Flow Engine 提供端到端的工作流能力，从前端操作视角分为以下功能域：

流程设计

流程列表、基本信息、流程画布编排、节点配置、表单设计、版本管理

待办中心

我的待办、我的已办、全部流程、发起流程，各列表点击进入详情/审批

流程中心

流程详情、流程记录轨迹、审批意见填写、8 种审批动作

移动审批

移动端首页、待办/已办/全部、流程申请与审批、流程详情

用户管理

用户与流程管理员维护

导入导出

流程定义 JSON 导入/导出，跨环境迁移

2. PC 端首页与导航

PC 端（app-pc）登录后进入首页，首页以导航卡片形式展示各功能入口，点击进入对应模块。

导航入口功能
流程设计进入流程定义列表，编排/配置/发布流程
待办中心我的待办 / 我的已办 / 全部流程 / 发起流程
流程演练Mock 实例管理，无真实库演练流程
用户管理用户与流程管理员维护
审批与流程详情页为隐藏路由，不直接出现在首页导航，而是由待办中心点击具体记录后携带参数进入。

3. 流程设计：流程列表

从首页进入 流程设计，首先看到流程定义列表，展示已存在的流程定义，可新建、编辑、删除、启停、导入导出。

3.1 操作步骤

  1. 首页点击 流程设计 进入流程列表
  2. 点击 新建 创建空流程定义（含默认开始/结束节点），进入设计器
  3. 或在列表点击已有流程的 编辑，加载流程定义进入设计器
  4. 列表支持启用/停用（/api/cmd/workflow/changeState）、删除（/api/cmd/workflow/remove）

4. 流程设计：基本信息与更多参数

进入设计器后，首先配置流程的基本信息（标题、编码、分类等），需要更深入配置时展开"更多参数"。

4.1 操作步骤

  1. 在设计器左侧/顶部切换到 基本信息 面板，填写流程标题、编码
  2. 需要配置流程级策略（如记录合并、超时、催办间隔、驳回配置等）时展开 更多参数
  3. 填写完成后保存（详见"版本管理与发布"章节）

4.2 基本信息属性

属性说明
流程标题流程的显示名称，便于识别与管理（必填）
流程备注对流程用途、规则的补充说明，可选填
流程编码流程的唯一编码，用于系统识别与接口调用（必填）
发起人范围通过脚本限定有权发起该流程的人员范围（必填）

4.3 更多参数属性

属性说明
干预配置 — 开启开启后，允许流程管理员协助用户操作该流程
催办 — 开启开启后，允许用户对当前审批人发起催办
催办间隔时间两次催办之间的最小间隔（单位：秒），仅在启用催办时生效

5. 流程设计：流程画布编排

流程画布是设计器的核心，基于 @flowgram.ai 固定布局编辑器，支持拖拽节点、连线编排流程。19 种节点从节点库拖入画布。

5.1 编排操作步骤

  1. 从节点库 拖拽 节点到画布，节点会按固定布局自动排列并连线
  2. 点击节点 选中，右侧/节点面板展示该节点配置（见下一节）
  3. 分支类节点（条件/包容/并行/人工）会自动带出对应 block 子节点，可调整分支数量
  4. 使用 迷你地图 快速定位，支持缩放/平移画布
  5. 编排完成进入节点面板逐个配置节点属性，再保存
每个节点的 meta 控制其行为：起始/结束节点禁止删除；分支节点禁止展开/复制等。这些约束由 node-registry 元数据定义，编辑器自动执行。

6. 流程设计：节点面板配置

选中画布上的节点后，节点面板展示该节点的配置，包括节点基本信息、按钮配置（启用哪些审批动作）、权限配置（表单字段权限、审批人加载方式等）。

6.1 节点可配置项

不同节点类型可配置的属性不同，以下按节点分组说明。各属性在节点面板标题旁的问号图标内均有相同提示，hover 可查看。

6.1.1 通用属性（全部节点）

属性适用节点说明
待办标题全部通过脚本设置待办消息的展示标题
优先级全部数字越小优先级越高；当多个分支或节点可同时执行时，按此排序
视图名称APPROVAL/HANDLE节点绑定的展示视图，用于自定义该节点的表单或界面展示
按钮配置（动作启用）APPROVAL/HANDLE勾选启用 8 种动作（PASS/REJECT/RETURN/TRANSFER/DELEGATE/ADD_AUDIT/SAVE/CUSTOM）
表单字段权限APPROVAL/HANDLE每字段必填 / 可编辑 / 只读 / 隐藏

6.1.2 审批 / 办理节点（APPROVAL / HANDLE）

属性说明
操作人设定方式脚本指定（SCRIPT，默认）/ 发起人设定（INITIATOR_SELECT）/ 审批人设定（APPROVER_SELECT，由当前审批人为下游指定）
设定人员范围 / 当前操作人发起人/审批人设定模式下，设定可选人员范围（留空表示不限范围，可选任意人）；脚本模式下通过脚本设定该节点操作人
多人审批策略循序提交（SEQUENCE，依次审批）/ 合并审核（MERGE，按会签比例合并判定）/ 任意审核（ANY，任一人审核即流转）/ 任意一人（RANDOM_ONE，随机指定一人审核）
会签通过比例合并审核时，通过人数达到该比例才视为该节点通过，取值 0~1 的小数（如 0.5 表示半数通过）
相同人员提交配置自动通过（AUTO_PASS，提交人与审批人同一人时自动通过）/ 手动审批（MANUAL_PASS，仍需手动审批）
启用合并审批开启后，当流程流转到该节点时，同一流程下的多条审批记录将自动合并到一起进行审批
审批意见是否必填开启后，审批人提交审批时必须填写审批意见
审批签名是否必填开启后，审批人提交审批时必须完成签名
启用撤回功能开启后，审批人在自己审批通过后可发起撤回，撤回后流程回到本节点重新审批（仅"已通过"状态且由原审批人发起时有效）
撤回方式撤回到当前节点（REVOKE_CURRENT，无论下级是否已审批均可撤回，下级记录全部作废）/ 撤回上级节点（REVOKE_NEXT，仅当下级节点尚未审批时才允许撤回，下级一旦审批完成则无法撤回）
重新提交配置跳转到撤回节点（RESUME，退回后直接跳转到上次撤回的节点继续审批）/ 从头逐级提交（CHAIN，退回后回到流程起点逐级重新提交）
超时类型自动提醒（REMIND，超时后发送提醒，不改变审批状态）/ 自动同意（PASS，超时后自动通过）/ 自动拒绝（REJECT，超时后自动拒绝）
超时时间(分钟)触发超时动作的等待时长，单位为分钟
异常配置当节点未能匹配到任何操作人时触发的脚本配置，可用于兜底处理

6.1.3 其他节点

属性适用节点说明
路由表达式ROUTER路由节点的判断脚本，根据返回值决定下一个流转节点
条件表达式CONDITION_BRANCH / INCLUSIVE_BRANCH条件分支的判断脚本，返回 true 时流程走该分支
子流程表达式SUB_PROCESS子流程触发的脚本，用于构造并在当前流程中发起一个子流程

触发表达式TRIGGER节点触发时执行的脚本，可用于动态计算或副作用处理
延迟时间类型DELAY延迟时间的计量单位（秒 / 分 / 时 / 天）

动作按钮在前端由 ActionFactory 工厂渲染。节点面板"按钮配置"中勾选的动作，会出现在该节点审批人的审批页底部按钮区。

7. 流程设计：表单设计

表单设计用于定义流程的业务表单字段（如请假的天数、事由）。可新增字段、编辑字段属性、调整排序。表单字段在发起流程与审批时由表单引擎渲染。

7.1 操作步骤

  1. 在设计器切换到 表单设计 面板
  2. 点击 新增字段，选择字段类型（string/integer/long/date/boolean 等，由 form-engine 注册）
  3. 填写字段标识、标题、是否必填，可拖拽调整字段顺序
  4. 字段定义后，在节点面板的"权限配置"中按节点控制每个字段的必填/可编辑/只读/隐藏

7.2 表单与子表属性

属性说明
表单名称主表的显示名称（必填）
表单编码主表的唯一编码，用于系统识别与数据存取（必填）
子表名称子表的显示名称（必填）
子表编码子表的唯一编码，用于系统识别与数据存取（必填）

7.3 字段属性（编辑字段弹窗）

属性说明
字段名称表单字段的显示名称（必填）
字段编码表单字段的唯一编码，用于数据存取，建议使用英文（必填）
字段类型表单字段的数据类型与展示组件（必填）
是否必填开启后，该字段为必填项
是否隐藏开启后，该字段在表单中不展示（仍可参与逻辑）
输入提示信息字段输入框内的占位提示文字
默认值字段新增时的初始默认值
提示信息字段标签旁的提示说明，hover 时显示
帮助提示字段下方的帮助说明文字
附加属性附加属性，主要用于一些复杂的表单控制界面
表单引擎（@coding-form/form-engine）支持通过 registerFormItems() 注册自定义字段类型，业务方可扩展自定义表单项。

8. 流程设计：版本管理与发布

流程定义支持多版本，便于不停机迭代。设计完成后保存为新版本并发布启用。

操作说明
保存新版本设计器保存时输入版本名，创建新版本并设为当前启用
切换版本版本列表中选择版本切换启用（/api/cmd/workflow/changeVersion），新发起流程走该版本
修改版本名/api/cmd/workflow/updateVersionName
删除版本/api/cmd/workflow/deleteVersion
导入/导出设计器导出 JSON 文件，或上传 JSON 导入（/api/cmd/workflow/import）

9. 待办中心（PC）

待办中心聚合 我的待办 / 我的已办 / 全部流程 三类记录，并提供 发起流程 入口。

9.1 操作步骤

  1. 首页点击 待办中心，默认进入"我的待办"
  2. 切换 Tab 查看 我的已办 / 全部流程（分别调用 /api/query/record/{done,list}）
  3. 在"我的待办"点击某条记录 → 进入流程审批页，填写审批意见后执行动作
  4. 在"我的已办/全部流程"点击记录 → 进入流程详情查看（只读）
  5. 点击 发起流程 → 选择流程定义 → 填写表单发起（见下节）

10. 发起流程与审批意见填写

待办中心的"发起流程"入口用于发起新的流程实例：选择流程定义后填写表单并提交。

审批人在"我的待办"中点击记录进入审批页，填写审批意见后选择动作（通过/拒绝/退回等）。

场景操作对应接口
发起流程选流程 → 填表单 → 提交（发起动作通常为开始节点的通过）POST /api/cmd/record/create
执行审批动作填审批意见/签名 → 选退回节点/转办人等 → 点击动作按钮POST /api/cmd/record/action
撤销/催办/删除发起人可撤销/催办；仅开始节点可删除/api/cmd/record/{revoke,urge,delete}

11. 流程中心：流程详情

流程详情展示一条流程实例的完整信息：业务表单、流程记录轨迹（已流转节点、当前节点、各节点的审批人与意见）。

11.1 流程记录轨迹

流程详情中的"流程记录/流程轨迹"展示每个节点的处理状态、处理人、处理时间、审批意见。审批人在审批页填写意见后，意见会记录在轨迹中。

三类核心记录职责
FlowRecord流程执行记录（TODO/DONE 状态）
FlowTodoRecord待办记录
FlowTodoMerge待办合并关系（开启合并时产生）
保存顺序：saveRecords → saveTodoMargeRecords → removeTodoMergeRecords，保证三者一致；删除顺序：合并关系 → 待办 → 流程记录。

12. 8 种审批动作操作

审批人在审批页底部看到由节点"按钮配置"决定的可执行动作按钮：

动作操作步骤
通过 PASS填写（必填）审批意见/签名 → 点击通过 → 流程向下一节点流转
拒绝 REJECT填写意见 → 拒绝 → 按配置结束流程或退回
退回 RETURN选择退回目标节点（只能退回历史 START/APPROVAL/HANDLE，不能退回当前/后续/结束节点）→ 填意见 → 退回
转办 TRANSFER选择转办人员 → 流程转给他人，自己不再处理
委派 DELEGATE选择委派人员 → 他人审批通过后流程回到自己
加签 ADD_AUDIT选择加签人员 → 增加会签人，会签完成后继续
保存 SAVE暂存当前表单与意见，不触发流转
自定义 CUSTOM触发节点配置的脚本/前端事件

12.1 人工分支选择

当流程流转到 MANUAL 节点，点击通过后引擎不立即流转，返回可选分支列表（responseType: MANUAL_NODE_SELECT）。前端弹出让用户选择走哪条分支，把选中的 manualNodeId 放入 advice 再次提交。

12.2 为下游节点指定审批人

对于 APPROVER_SELECT 模式的下游节点，审批人在通过时需在 operatorSelectMap 中为其指定审批人；INITIATOR_SELECT 模式则由发起人在发起流程时指定。

13. 用户管理

用户管理用于维护系统用户与流程管理员（isFlowManager），审批人由 FlowOperatorGateway 按用户体系加载。

流程管理员（isFlowManager=true）可进行流程强制干预，并可作为 Mock 模式的创建者（/api/cmd/workflow/mock 需流程管理员权限）。

14. 移动端操作

app-mobile 提供精简的移动审批能力（无流程设计器，设计在 PC 完成）。移动端首页与待办列表如下：

14.1 流程申请与审批意见填写

移动端发起流程申请时填写表单，提交后生成流程记录；审批人在待办点击进入后填写审批意见。

14.2 流程审批与详情

点击待办记录进入流程审批，可查看流程详情与流程记录，并执行动作。

14.3 移动端操作步骤

  1. 登录 → 进入首页，导航到待办中心
  2. 待办列表（待办/已办/全部 切换）点击卡片 → 进入流程审批页（携带 recordId、workflowCode）
  3. 查看流程详情与流程记录，填写审批意见
  4. 点击 通过/拒绝/退回 等动作按钮执行
  5. 审批完成自动返回列表

14.4 移动端特性

  • 使用 antd-mobile 组件，flow-mobile-ui 提供 TextIcon 等封装
  • ActionFactory 移动版返回适配移动交互的动作按钮
  • 列表复用同一组件，按 type（todo/done/notify/list）切换 API
  • 通过 EventBus 监听事件联动

15. 常见问题

问题解答
审批时按钮只有部分？动作按钮由节点"按钮配置"决定，在设计器节点面板勾选所需动作
退回节点选不到目标？退回目标只能是历史 START/APPROVAL/HANDLE 节点，且不能是当前节点或其后续节点、不能是结束节点
表单字段不显示？字段权限设为"隐藏"时不显示；字段类型需在 registerFormItems 中注册
发起流程选不到流程？流程定义可能被停用，或没有已启用版本
审批人不对？检查节点"审批人加载方式"与脚本；INITIATOR_SELECT 需发起时指定，APPROVER_SELECT 需上游指定
PC 与移动端功能差异？PC 含流程设计与完整审批；移动端仅审批与查询，设计在 PC 完成
如何不登录调试？请求加 ?operatorId=xxx 指定操作者；用 ?mockKey=xxx 走 Mock 实例
流程定义如何迁移？设计器导出 JSON → 目标环境导入（/api/cmd/workflow/import）
自定义按钮怎么做？通过 ActionFactory 注册 CUSTOM 类型组件 + 节点按钮配置启用 CUSTOM + 配置触发脚本
