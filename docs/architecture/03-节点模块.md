# 03 - 节点模块

## 模块概述

**包路径**: `com.codingapi.flow.node`

节点是流程执行的基本单元，定义了流程中各个处理步骤的行为。

## 核心接口

### IFlowNode

```java
package com.codingapi.flow.node;

/**
 * 流程节点接口
 */
public interface IFlowNode {

    /**
     * 获取节点ID
     */
    String getNodeId();

    /**
     * 获取节点名称
     */
    String getNodeName();

    /**
     * 获取节点类型
     */
    String getNodeType();

    /**
     * 获取节点顺序
     */
    int getOrder();

    /**
     * 节点验证
     * @return true 验证通过
     */
    boolean validate();

    /**
     * 处理流程
     * @param context 流程上下文
     */
    void handle(FlowContext context);
}
```

### IBlockNode

```java
package com.codingapi.flow.node;

/**
 * 块节点接口 - 包含子节点的节点

```

### IDisplayNode

```java
package com.codingapi.flow.node;

/**
 * 显示节点接口
```

## 节点类型

| 节点类型 | 类名 | 说明 |
|----------|------|------|
| 开始节点 | `StartNode` | 流程入口 |
| 结束节点 | `EndNode` | 流程终点 |
| 审批节点 | `ApprovalNode` | NODE_TYPE = "Approval" |
| 办理节点 | `HandleNode` | NODE_TYPE = "Handle" |
| 路由节点 | `RouterNode` | NODE_TYPE = "Router" |
| 条件分支 | `ConditionBranchNode` | 条件判断分支 |
| 并行分支 | `ParallelBranchNode` | 并行执行分支 |
| 并行节点 | `ParallelNode` | 聚合并行分支 |
| 子流程 | `SubProcessNode` | 调用子流程 |
| 触发器节点 | `TriggerNode` | 触发外部事件 |
| 手动节点 | `ManualNode` | 人工触发节点 |

## 节点工厂

### NodeFactory

```java
package com.codingapi.flow.node;

public class NodeFactory {

    public static IFlowNode createNode(String nodeType) {
        return switch (nodeType) {
            case "Start" -> new StartNode();
            case "End" -> new EndNode();
            case "Approval" -> new ApprovalNode();
            case "Handle" -> new HandleNode();
            case "Router" -> new RouterNode();
            case "ConditionBranch" -> new ConditionBranchNode();
            case "ParallelBranch" -> new ParallelBranchNode();
            case "Parallel" -> new ParallelNode();
            case "SubProcess" -> new SubProcessNode();
            case "Trigger" -> new TriggerNode();
            case "Manual" -> new ManualNode();
            default -> throw new FlowException("Unknown node type: " + nodeType);
        };
    }
}
```

## 节点层级结构

```
StartNode
    │
    ▼
ApprovalNode
    │
    ▼
RouterNode
    ├──► Condition ──► ParallelNode
    │                    ├──► ParallelBranchNode ──► ApprovalNode
    │                    └──► ParallelBranchNode ──► ApprovalNode
    │
    └──► HandleNode
             │
             ▼
         EndNode
```

## 节点配置结构

节点配置以 JSON 形式存储在 `WorkflowVersionEntity.nodeMetas` 字段中：

```json
{
  "nodeId": "node_001",
  "nodeType": "Approval",
  "nodeName": "部门经理审批",
  "order": 1,
  "config": {
    "approvalType": "single",
    "allowDelegate": true,
    "timeoutStrategy": {...}
  }
}
```
