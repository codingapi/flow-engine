# 06 - 脚本模块

## 模块概述

**包路径**: `com.codingapi.flow.script`

脚本模块提供 Groovy 脚本运行时支持，允许在流程中执行自定义业务逻辑。

## 包结构

```
com.codingapi.flow.script
├── node/                    # 节点脚本
│   ├── ConditionScript.java
│   ├── NodeTitleScript.java
│   ├── OperatorMatchScript.java
│   ├── RouterNodeScript.java
│   ├── SubProcessScript.java
│   ├── TriggerScript.java
│   ├── OperatorLoadScript.java
│   └── ErrorTriggerScript.java
├── action/                  # 动作脚本
│   ├── CustomScript.java
│   └── RejectActionScript.java
└── runtime/                 # 运行时
    ├── ScriptRuntimeContext.java

    └── FlowScriptContext.java
```

## 脚本类型

### 节点脚本

| 脚本类 | 说明 | 使用场景 |
|--------|------|----------|
| `ConditionScript` | 条件判断脚本 | 路由节点条件判断 |
| `NodeTitleScript` | 节点标题脚本 | 动态生成节点显示标题 |
| `OperatorMatchScript` | 操作者匹配脚本 | 动态匹配审批人 |
| `RouterNodeScript` | 路由节点脚本 | 自定义路由逻辑 |
| `SubProcessScript` | 子流程脚本 | 子流程参数映射 |
| `TriggerScript` | 触发器脚本 | 触发外部系统 |
| `OperatorLoadScript` | 操作者加载脚本 | 批量加载操作者 |
| `ErrorTriggerScript` | 错误触发脚本 | 异常情况处理 |

### 动作脚本

| 脚本类 | 说明 |
|--------|------|
| `CustomScript` | 自定义动作脚本 |
| `RejectActionScript` | 拒绝动作脚本 |

## 运行时上下文

### ScriptRuntimeContext

```java
package com.codingapi.flow.script.runtime;

/**
 * Groovy 脚本运行时上下文
 */
public class ScriptRuntimeContext {

    private static final ScriptRuntimeContext INSTANCE = new ScriptRuntimeContext();

    private GroovyClassLoader classLoader;
    private IBeanFactory beanFactory;

    private ScriptRuntimeContext() {
        this.classLoader = new GroovyClassLoader();
    }

    public static ScriptRuntimeContext getInstance() {
        return INSTANCE;
    }

    /**
     * 执行脚本
     */
    public Object executeScript(String scriptContent, FlowScriptContext context) {
        // 1. 编译脚本
        Class<?> scriptClass = classLoader.parseClass(scriptContent);

        // 2. 创建脚本实例
        GroovyObject script = (GroovyObject) scriptClass.getDeclaredConstructor().newInstance();

        // 3. 设置上下文变量
        Binding binding = new Binding();
        binding.setVariable("context", context);
        binding.setVariable("workflow", context.getWorkflow());
        binding.setVariable("record", context.getFlowRecord());
        binding.setVariable("operator", context.getOperator());

        // 4. 执行脚本
        script.setBinding(binding);
        return script.invokeMethod("run", null);
    }
}
```

### FlowScriptContext

```java
package com.codingapi.flow.script.runtime;

/**
 * 流程脚本上下文
 */
public class FlowScriptContext {

    private Workflow workflow;           // 工作流
    private FlowRecord flowRecord;       // 流程记录
    private IFlowOperator operator;     // 当前操作者
    private Map<String, Object> params;  // 额外参数
}
```

### IBeanFactory

```java
package com.codingapi.flow.script.runtime;

/**
 * Bean 工厂接口 - 允许脚本访问 Spring Bean
 */
public interface IBeanFactory {

    /**
     * 获取 Bean
     */
    <T> T getBean(Class<T> beanClass);

    /**
     * 获取 Bean
     */
    Object getBean(String beanName);
}
```

## 脚本示例

### 条件判断脚本

```groovy
// ConditionScript 示例
def result = false
if (context.formData.amount > 10000) {
    // 金额大于1万需要总监审批
    result = true
}
return result
```

### 节点标题脚本

```groovy
// NodeTitleScript 示例
def title = "请假审批"
if (context.formData.days > 3) {
    title = "长时间请假审批 - " + context.formData.days + "天"
}
return title
```

### 操作者匹配脚本

```groovy
// OperatorMatchScript 示例
def operators = []
def deptId = context.operator.deptId
// 查询同部门经理
def managers = userService.getUsersByRoleAndDept("MANAGER", deptId)
return managers
```

## 脚本执行流程

```
节点处理
    │
    ▼
加载脚本配置
    │
    ▼
获取脚本类型
    │
    ▼
编译 Groovy 脚本
    │
    ▼
创建脚本上下文 ──► 设置绑定变量
    │
    ▼
执行脚本
    │
    ├──► 成功 ──► 返回结果
    │
    └──► 异常 ──► 执行错误处理脚本
                      │
                      ▼
                 记录错误日志
```
