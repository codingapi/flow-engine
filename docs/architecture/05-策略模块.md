# 05 - 策略模块

## 模块概述

**包路径**: `com.codingapi.flow.strategy`

策略模块采用策略模式，封装节点和工作流的各类行为逻辑，实现策略的可插拔配置。

## 节点策略

### INodeStrategy 接口

```java
package com.codingapi.flow.strategy.node;

/**
 * 节点策略接口
 */
public interface INodeStrategy {

    /**
     * 获取策略类型
     */
    String getType();

    /**
     * 执行策略
     * @param context 策略上下文
     */
    void execute(StrategyContext context);
}
```

| 策略类 | 类型标识 | 说明 |

| `SameOperatorAuditStrategy` | "sameOperatorAudit" | 同操作者审批策略 |
| `RecordMergeStrategy` | "recordMerge" | 记录合并策略 |
| `ResubmitStrategy` | "resubmit" | 重新提交策略 |
| `AdviceStrategy` | "advice" | 审批意见策略 |
| `ErrorTriggerStrategy` | "errorTrigger" | 错误触发策略 |
| `NodeTitleStrategy` | "nodeTitle" | 节点标题策略 |
| `FormFieldPermissionStrategy` | "formFieldPermission" | 表单字段权限策略 |
| `RevokeStrategy` | "revoke" | 撤销策略 |
| `RouterStrategy` | "router" | 路由策略 |
| `TriggerStrategy` | "trigger" | 触发策略 |
| `DelayStrategy` | "delay" | 延迟策略 |
| `OperatorLoadStrategy` | "operatorLoad" | 操作者加载策略 |

### TimeoutStrategy 示例

```java
package com.codingapi.flow.strategy.node;

public class TimeoutStrategy implements INodeStrategy {

    public static final String TYPE = "timeout";

    @Override
    public String getType() {
        return TYPE;
    }

    @Override
    public void execute(StrategyContext context) {
        // 1. 获取超时配置
        TimeoutConfig config = context.getConfig(TimeoutConfig.class);

        // 2. 检查是否超时
        long elapsed = System.currentTimeMillis() - context.getStartTime();
        if (elapsed > config.getTimeoutMillis()) {
            // 3. 执行超时处理
            handleTimeout(context);
        }
    }
}
```

## 策略工厂

### NodeStrategyFactory

```java
package com.codingapi.flow.strategy.node;

public class NodeStrategyFactory {

    private static final Map<String, INodeStrategy> STRATEGIES = new ConcurrentHashMap<>();

    static {
        register(new TimeoutStrategy());
        register(new MultiOperatorAuditStrategy());
        register(new SameOperatorAuditStrategy());
        register(new RecordMergeStrategy());
        register(new ResubmitStrategy());
        register(new AdviceStrategy());
        register(new RevokeStrategy());
        register(new RouterStrategy());
        register(new TriggerStrategy());
        register(new DelayStrategy());
    }

    public static void register(INodeStrategy strategy) {
        STRATEGIES.put(strategy.getType(), strategy);
    }

    public static INodeStrategy getStrategy(String type) {
        INodeStrategy strategy = STRATEGIES.get(type);
        if (strategy == null) {
            throw new FlowException("Unsupported strategy type: " + type);
        }
        return strategy;
    }
}
```

## 策略配置结构

节点策略配置以 JSON 形式存储：

```json
{
  "nodeId": "node_001",
  "strategies": [
    {
      "type": "timeout",
      "config": {
        "timeoutMillis": 86400000,
        "handleType": "autoPass"
      }
    },
    {
      "type": "multiOperatorAudit",
      "config": {
        "approvalCount": 2
      }
    }
  ]
}
```

```
     │

     │

     │

             │

```
