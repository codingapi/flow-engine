## viewTitle 回退逻辑移除

移除了 viewTitle（视图标题）为空时默认回退到节点名称（name）的逻辑，使 viewTitle 在未设置时保持真正的空值。

### 修改内容：

**后端 Java：**
- `BaseAuditNode.java`：移除 `getViewTitle()` 的回退逻辑（`StringUtils.hasText(viewTitle) ? viewTitle : getName()`），改为 `@Getter` 直接暴露字段；`toMap()` 中 `map.put("viewTitle", viewTitle)` 直接使用字段而非 `getViewTitle()`
- `StartNode.java`：同样移除回退逻辑，改为 `@Getter`；移除不再使用的 `StringUtils` import；`toMap()` 直接使用 `viewTitle` 字段
- `FlowContent.java`：无需修改，`viewTitle` 是 String 类型天然可 null

**前端 TypeScript：**
- `node.ts` `toItemRender()`：`node.viewTitle || node.name` → `node.viewTitle`
- `node.ts` `toDataItem()`：`data?.viewTitle || data?.title` → `data?.viewTitle`
- `initial-data.ts`：移除开始节点的 `viewTitle: '开始节点'` 默认值
- `flow-approval.ts`：`viewTitle: string` → `viewTitle?: string`（改为可选）

**单元测试：**
- `NodeFactoryTest.shouldFallbackViewTitleWhenImportOldNodeData` → `shouldKeepViewTitleNullWhenNotSet`：验证 viewTitle 未设置时确实为 null
